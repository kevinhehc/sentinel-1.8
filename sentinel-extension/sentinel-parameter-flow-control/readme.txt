热点参数限流（Hotspot Parameter Flow Control） 是为了防止某些 热点参数值 被高频访问，导致系统过载。
	•	普通的限流（FlowRule）只关心整个资源的 QPS。
	•	热点规则（ParamFlowRule） 会区分不同的参数值，对“单个热点参数”做限流。

这样可以避免 局部热点 导致全局被限流。

-------------------------------------------------------------------------------------------
场景 1：商品详情页
	•	系统接口：/item/getDetail
	•	参数：itemId

如果某个爆款商品（比如 itemId=12345）在短时间被大量请求，就会拖垮整个商品详情接口。
你可以设置热点规则：
	•	对 /item/getDetail 按 itemId 维度做限流。
	•	itemId=12345 每秒最多 100 次，其它 itemId 不受影响。

这样，热点商品被限流时，其他商品还能正常访问，系统不会被热点冲垮。

-------------------------------------------------------------------------------------------
场景 2：用户登录接口
	•	系统接口：/user/login
	•	参数：userId

如果某个用户被恶意攻击（疯狂请求），那么整个 /user/login 接口的 QPS 会被顶满。
可以设置热点规则：
	•	针对 userId 做限流，比如某个用户 ID 每秒最多 10 次请求。
	•	这样能避免单个用户拖垮所有用户的登录接口。
